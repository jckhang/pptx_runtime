## 第4页：角色分工矩阵
- 核心内容：主代理与子代理的职责对比表
- 主代理职责：需求分析、视觉设计（design.md）、内容规划（outline.md）、.pptd 主文件生成、质量检查
- 子代理职责：.page 页面文件构建（仅执行，不设计）
- 关键规则：子代理禁止使用 create_subagent/task 工具进一步派发任务

## 第5页：代理数量控制策略
- 决策阈值：单 PPT ≤ 20 页 → 主代理自行完成；> 20 页 或 多个 PPT → 启用子代理
- 子代理分组策略：按页面类型分组
  - 子代理 A：封面 + 目录 + 章节过渡页 + 结束页
  - 子代理 B/C/D：内容页（每组最多 20 页）
- 信息流转：主代理设计 → 子代理执行 → 主代理检查

## 第7页：PPTD 设计哲学
- 核心定位：PPTD = Presentation Description，面向 AI 设计的演示文稿描述语言
- 四大设计原则：
  1. YAML 语法：人类可读、AI 易写，避免 XML 复杂度
  2. 自包含页面：每个 .page 独立描述，不依赖 Master 继承
  3. 所见即所得：元素位置、样式、内容一目了然
  4. 主题系统：通过 $ 引用实现统一管理
- 关键约束：禁止直接操作 .pptx，用户通过前端"导出"按钮转换

## 第8页：PPTD 功能特性
- 六大元素类型：
  1. Text（富文本，支持 HTML + LaTeX）
  2. Shape（177 种形状）
  3. Image（图片填充、裁剪）
  4. Icon（Font Awesome 7.x）
  5. Table（合并单元格、主题样式）
  6. Chart（8 种图表类型）
- 三大样式系统：Fill（纯色/渐变/图片）、Border、Shadow
- 文件结构：.pptd 主入口 + pages/ 目录 + images/ 目录

## 第9页：PPTD 文件结构示例

- 路径规则：pages 列表使用相对路径，image src 使用绝对路径或 URL

## 第15页：Checker 检查维度
- 两大问题类别：
  - Error（转换失败，必须修复）：语法错误、字段缺失、ID 重复
  - Warning（视觉问题，必须修复）：文本溢出、文本遮挡、文本漂移、文本欠填充、元素越界
- 运行方式：全量检查 / 指定页面检查（--pages 参数）
- 检查维度表格：格式检查、数据验证、布局检测、文本框内容检测

## 第16页：修复策略
- 修复优先级：Error → Warning
- 并行修复原则：单次响应中尽可能并行调用 edit_file
- 文本溢出修复顺序：
  - 高度溢出：精简文字 → 减小字号/行距 → 扩大文本框
  - 宽度溢出：精简文字 → 开启自动换行 → 调整布局
- 修复注意事项：保持边距、不动公共元素位置、确保内容同步调整

## 第18页：五大设计原则
- 关注点分离：设计（主代理）与构建（子代理）分离
- 格式抽象：PPTD 作为 AI 友好的中间层，屏蔽 OOXML 复杂度
- 质量内建：Checker 自动化检测，Error/Warning 分类处理
- 可扩展性：3×3 模式矩阵覆盖绝大多数 PPT 场景
- 并行效率：子代理按页面类型分组并行，修复操作批量并行
